# cli-feegrant characterization

- Base SHA: `7aa166f1dde72cbc91e399ce6966fac5a24a3b39`
- Ownership: `complexity-characterization-engineer`; writable scope limited to characterization tests under `cmd/*_test.go` and this document. `cmd/feegrant.go`, campaign state, harness files, and Git state remained read-only.
- Functions/scores: `feegrantConfigureBasicCmd` `26/45` and `feegrantBasicGrantsCmd` `7/13` (cyclomatic/cognitive). The gate permits scores at most 10; only scores greater than 10 violate.
- Observable contracts:
  - Configure Use, minimum-one positional argument behavior, defaults, and mutual exclusion are frozen. Number of Grantees defaults to 10; Delete, Overwrite Granter, and Overwrite Grantees default false; Grantees is an empty slice; Gas and Memo default empty. Cobra rejects combinations among Number of Grantees, Grantees, and Delete before RunE or broadcast.
  - Both commands resolve the chain before asserting `*cosmos.CosmosProvider`; missing-chain guidance and the non-Cosmos provider error remain exact. Configure accepts more than two positional arguments because it uses `MinimumNArgs(1)`, while Basic Query accepts one or two.
  - Default granter uses the provider key. A valid Bech32 address not present in the keyring is treated as external, and an external granter without explicit pre-authorized Grantees fails before config mutation or network work. Explicit internal granter names resolve through the keyring.
  - Delete logs `Deleting feegrant configuration`, reloads under the config lock, clears FeeGrants on disk, performs no queries/broadcast, and returns before Memo/Gas handling.
  - Overwrite Granter mutates the originally captured provider before the late Memo getter. When RunE receives a command without a Memo flag, it returns the exact getter error; captured memory contains the new granter while the first locking save still contains the old granter. This freezes the partial-side-effect ordering.
  - Explicit Grantees preserve input order. With Grantees `grantee1,grantee2`, Memo `characterized memo`, and Gas `456789`, EnsureBasicGrants queries existing grants, builds two BasicAllowance MsgGrantAllowance messages in that order, signs and broadcasts one transaction, waits through the Tx GetTx query, and then records latest height 91.
  - The decoded TxRaw/TxBody/AuthInfo preserves Memo, Gas Limit, exactly two messages, BasicAllowance type URLs, the default-key granter address, and each grantee key address in order. Logs preserve two `Creating feegrant` entries with matching fields, followed by `Feegrant succeeded` and `feegrant configured` with height 91.
  - A broadcast error is wrapped exactly as `error writing grants on chain: 'broadcast failed'`. FeeGrant configuration and Managed Grantees have already been persisted at that point, but BlockHeightVerified remains zero and no success/configured log is emitted.
  - Basic Query's optional granter argument is currently ignored: RunE uses `args[0]`, the chain name, as the key/address. The fake verifies the resulting chain-named key address is sent to AllowancesByGranter even when a second argument is supplied. Returned grants are logged in response order with Granter, Grantee, and rendered Allowance. Pagination flags retain defaults but are not passed by this command because it always calls the provider with a nil paginator.
  - Configure's numeric Gas parsing reaches the transaction AuthInfo unchanged. Latest Height is queried only after successful grant establishment and persisted after the transaction wait. These commands expose no duration flag; allowance expiration/duration parsing belongs to the Cosmos feegrant implementation, not these two CLI targets.
- Fake/runtime seams: tests instantiate the real concrete CosmosProvider required by the production type assertion, attach an in-memory keyring, and provide a `cclient.ConsensusClient` fake. The fake answers Feegrant, Auth Account, and Tx GetTx ABCI/gRPC queries, records asynchronous broadcast bytes, and returns deterministic height 91. Tests decode the actual signed transaction. Config reload may log `No backup RPCs defined`; assertions filter only that provider-liveliness noise. No RPC or public network is used.
