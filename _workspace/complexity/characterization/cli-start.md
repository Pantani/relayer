- Functions/scores: `cmd/start.go:startCmd` `17/33`, `cmd/start.go:setupDebugServer` `12/13`, and `cmd/flags.go:getAddInputs` `11/10` (cyclomatic/cognitive). The campaign gate allows scores at most 10. `setupMetricsServer` `9/10` is already valid and is not a production target.
- Observable contracts:
  - `getAddInputs` reads `file`, `url`, `force-add`, then `testnet`, preserving values read before an accessor error. File plus URL is rejected before either Testnet conflict, and every semantic-conflict result is cleared to zero values. File or URL combined with Testnet is rejected; Force Add alone and Testnet alone are accepted.
  - `setupDebugServer` resolves configuration in this order: `debug-listen-addr`, deprecated `api-listen-addr` fallback, deprecated `--debug-addr`, then `--debug-listen-addr`. It reads the three flags in the order deprecated address, listen address, enable switch. A non-empty deprecated flag enables the server without the enable switch; the new listen flag still wins as the final address.
  - Deprecated configuration and flag warnings, disabled/missing-address messages, enabled/security messages, and the listen failure log retain their original text and order. Listen errors wrap the selected address. The incoming `err` parameter is ignored once flag access begins.
  - `startCmd` retains its use, alias and short description, along with defaults for processor, block history, maximum messages, debug/metrics enablement and addresses, memo, and stuck-packet controls.
  - Named-path selection happens before chain lookup and panics for an unknown path through `Paths.MustGet`. Both one explicit configured path and all configured paths collect chain IDs before returning the exact missing-chain error. Existing `cmd/start_test.go` coverage additionally freezes successful disabled/enabled debug and metrics flows, address overrides, HTTP availability, missing-address behavior, and user-visible log messages.
