Existing relayer processor tests remain responsible for runtime packet/ack lifecycle internals.
  - Successful `CreateOpenChannels` and `CloseChannel` enter event processors. Tests freeze their command-level inputs and all deterministic short-circuit/provider boundaries, not processor internals.
