- Observable contracts:
  - Command structure is frozen for Use, aliases, short descriptions, accepted argument counts, and JSON/YAML/File/URL/Force Add/Testnet defaults. List and Registry List accept no positional arguments, Show accepts exactly one, and Add accepts zero or more.
  - List reads `--json` before `--yaml`. With no configured chains it writes the exact warning to stderr before rejecting a simultaneous JSON/YAML request. Plain output retains its index/column widths and key, balance, and path glyphs. JSON and YAML retain their exact provider-map envelopes and trailing newlines.
  - Show checks chain existence before reading `--json`, returns the exact missing-chain guidance, defaults to YAML, and retains exact JSON/YAML provider envelopes and trailing newline behavior.
  - Registry List reads JSON then YAML before any network access. It fetches and filters the GitHub tree before checking the JSON/YAML conflict, so even the invalid combined-format request performs one registry request. Plain output preserves registry order, JSON is compact, YAML includes its existing extra newline, hidden directories and blobs are filtered, and transport errors retain the HTTP request context.
  - Add reads all four input flags before checking whether configuration is initialized. File plus URL conflict therefore wins over config initialization and filesystem/network work. A file with no explicit name derives the name from the basename; more than one file name is rejected without mutation. URL mode requires exactly one name before issuing a request. Successful File and URL additions mutate runtime state and persist the provider into `config.yaml` with silent stdout/stderr.
  - Registry batch processing skips an exact map-key duplicate without a request, warns, and continues. Retrieval/config-generation failures are logged and accumulated while later entries still run. Testnet changes the raw registry path to `/testnets/<name>/chain.json`. Successful insertion uses the registry's canonical `chain_name` as the config map key while the summary's `added` list retains the requested name.
  - With zero registry RPC endpoints, `forceAdd=false` logs `Endpoints queried`, records config generation failure, adds nothing, and still returns nil after its status summary. `forceAdd=true` still emits `Endpoints queried`, then accepts an empty primary/backup RPC set and adds the chain. Final `Config update status` preserves the `added`, `failed`, and `already existed` fields.
- Network seams: tests replace `http.DefaultTransport` only for sequential test scope and restore it with cleanup. This intercepts both `api.github.com` tree requests and `raw.githubusercontent.com` registry requests. Registry fixtures include a fee token and no RPC endpoints, so no public request or RPC health probe occurs. URL-mode coverage uses `httptest.Server`.
